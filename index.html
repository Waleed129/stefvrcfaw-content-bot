Content Posting API Integration

Discord-based video submission, review, and publishing workflow.

FBI VRP Content Bot is a Discord bot that allows authorized users to submit videos, captions, and hashtags from Discord for moderation review and possible publishing to a connected TikTok account using the official Content Posting API.

Users can start the account connection process from this page, review the legal pages, and understand how the submission and publishing workflow operates.

Core workflow

  • Authorized Discord users submit a video request.
  • Moderators review captions, hashtags, and file details.
  • Requests can be approved, rejected, or edited before publishing.
  • Approved content is uploaded and published to a connected TikTok account.
  • Status and operational logs are tracked for moderation and troubleshooting.

Account connection

To use the publishing workflow, an authorized user must connect an account through the login entry point below. This starts the OAuth authorization process and allows the service to receive the permissions required for the Content Posting API.

Login / Connect account
Access to the publishing workflow is controlled. Connecting an account does not guarantee that every submitted video will be published. All submissions may still require moderation approval and must comply with platform and server rules.

What this service does

Submission

Users can submit video files together with captions and hashtags from Discord through a controlled workflow.

Moderation

Authorized staff can review each request, inspect content details, edit text fields, and decide whether to approve or reject the submission.

Publishing

Once approved, the bot processes the request and sends the content through the official API, then tracks publish status and logs outcomes.

How it works

1

Connect

An authorized user starts the login flow and connects the required account permissions.

2

Submit

A user uploads a supported video file and includes caption text or hashtags inside Discord.

3

Review

The request is sent to a designated approval workflow where authorized staff can inspect it.

4

Publish

Approved requests are uploaded to the connected TikTok account and tracked until completion or failure.

Key service characteristics

Permission-controlled

The workflow is designed for controlled server environments with role-based access, moderation review, and approval restrictions.

Operational logging

Request states, approvals, failures, timestamps, and related operational events may be logged for troubleshooting and moderation history.

API-based publishing

The service relies on official API access and is subject to platform rules, availability, restrictions, and review requirements.

Publication is not guaranteed. Requests may be delayed, rejected, limited, or fail because of moderation decisions, service settings, third-party platform restrictions, or API-related conditions.
